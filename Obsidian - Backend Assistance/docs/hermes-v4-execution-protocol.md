# HERMES V4 执行协议

## 目标

规定 HERMES 执行 V4 工程时的批次、停止、报告、提交与回滚方式。

## 执行批次

1. Batch 0 仓库体检。
2. Batch 1 文档规范层。
3. Batch 2 模板层。
4. Batch 3 CSS 视觉层。
5. Batch 4 安全写入层。
6. Batch 5 生成器层。
7. Batch 6 Demo 课程层。
8. Batch 7 测试层。
9. Batch 8 README 与交付报告。

## 每批输出

- 本批次目标。
- 新增文件。
- 修改文件。
- 风险点。
- 自测结果。
- 下一批次建议。

## 停止协议

用户输入“停止/暂停/打断/不要继续”时，立即停止新写入，报告当前分支、最后提交、已完成 Batch 和恢复点。

## 执行要求

- 每批次尽量单独 commit。
- 出现高风险隐私/素材外溢时停止后续批次。
- 不执行正式 vault 写入。

## 验收标准

- 每批次都有可追溯报告或提交记录。
- 出错后可以通过 git revert 回滚。

## 安全边界

- 本规范只服务辅助仓库，不要求写入正式 Obsidian vault。
- 文档中不得包含真实课程正文、原始素材、转写全文、OCR 全文、token 或 API key。
- 所有示例路径必须使用占位符或 demo 路径，不能依赖用户正式库路径。
