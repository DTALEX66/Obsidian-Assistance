紫晶阅读 · 关键摘记

#course #evidence #obsidian #project-atlas
知识只有进入项目、证据和复盘链路，才会从“看过”变成可执行资产。

1. 阅读不是收集，而是建证据

课程阅读页要保持两件事同时发生：左手不打断阅读，右手把关键片段压缩成可以归档、复核、迁移的 Evidence Card。

  • 每一段摘记都要能回答“它支撑哪个判断”。
  • 不确定的内容先进入草稿，不伪造成结论。
  • 可迁移字段优先：来源、主题、证据强度、下一步。

2. 四段归档链路

2.1 提炼概念

用一句话写出这段内容解决的问题。
✓绑定当前课程来源，避免孤立卡片。
标出进入 Project Atlas 的行动位置。

2.2 写成证据

  • 概念负责压缩理解，证据负责支撑判断。
  • 证据卡不追求长文，追求可检索、可复盘、可引用。
  • 进入矩阵后再决定是否提升为项目资产。
Key Idea: TALOS 的阅读界面必须像 Obsidian 原生工作区一样组织信息，但数据结构要能迁移成 Markdown frontmatter、块引用和插件 ItemView。 
```dataview
TABLE file.link, evidence.status, evidence.strength
FROM #course-reading
WHERE evidence.ready = true
SORT evidence.strength DESC
```
