# 一比一拆解执行方案

## 0. 总结判断

这组图不是单一皮肤，而是一套 Obsidian Knowledge OS：

1. 首页操作台 Dashboard
2. 双栏笔记 / 阅读工作区
3. Canvas 知识画布 / 项目脑图
4. Review + AI Assistant 复习系统
5. 紫晶主题视觉资产板 / 组件库

落地方式分三层：

| 层级 | 能做到什么 | 技术方式 | 还原度 |
|---|---|---|---|
| A 层：无需开发 | 主题色、卡片、首页、模板、Canvas、Dataview/Bases 数据视图 | CSS Snippet + Markdown + 插件 | 75%-85% |
| B 层：轻开发 | 自定义命令、快捷按钮、右侧 AI 面板、状态栏统计 | Obsidian 插件 | 85%-95% |
| C 层：深开发 | 顶栏、侧栏、图标、工作区布局完全复刻 | 自研主题 + 插件 + workspace layout | 95%+ |

本包先给 A 层完整落地，并为 B/C 层预留结构。

## 1. 图一：Dashboard 首页操作台

### 视觉结构

- 画布比例：16:9 宽屏工作台。
- 左侧：Vault 导航与最近 Vault。
- 顶部：Tab 栏 + 全局搜索 + 状态图标。
- 中间三列：欢迎区、最近笔记、任务、阅读队列、快速记录、插件状态、活动图表。
- 右侧：Calendar、Backlinks、Tags、Vault Stats。
- 底部：Live Graph 状态栏。

### Obsidian 落地

| 图中模块 | Obsidian 对应 | 文件/插件 |
|---|---|---|
| Welcome Hero | Markdown HTML 卡片 | `00_操作台/00-Dashboard.md` |
| Recent Notes | Dataview 查询 | Dataview |
| Tasks | Tasks 查询 | Tasks |
| Reading Queue | frontmatter + Dataview/Bases | Book 模板 |
| Quick Capture | Inbox + 快捷新建 | Templater / QuickAdd 可选 |
| Plugin Status | 手工表或插件页 | Markdown 表 |
| Activity Chart | DataviewJS 可扩展 | DataviewJS 可选 |
| Calendar | Calendar 插件 / Daily Notes | Calendar |
| Backlinks | Obsidian 核心 Backlinks | 核心插件 |
| Tags | Tag Pane | 核心插件 |

### 页面布局参数

- 主布局：`grid-template-columns: 1.1fr 1fr .72fr`
- 卡片间距：16px
- 卡片圆角：16px
- 卡片背景：`rgba(20,24,42,.78)`
- 主高光：`#8B5CF6`
- 辅助高光：`#38BDF8`
- 字体：Inter / Segoe UI / Microsoft YaHei

### 执行动作

1. 把 `00_操作台/00-Dashboard.md` 放到 Vault。
2. 启用 CSS snippet。
3. 安装 Dataview、Tasks、Calendar。
4. 把 Dashboard 设为启动页：首页插件可用 Homepage；不装也可以手动固定标签页。

## 2. 图二：双栏笔记 / 阅读工作区

### 视觉结构

- 左侧文件树：知识库目录，当前笔记高亮。
- 中间：编辑模式，保留 Markdown 标记。
- 右侧：阅读预览模式，展示最终排版。
- 最右侧：Outline、Properties、Linked Mentions。
- 底部：字数、反链、Graph 状态。

### Obsidian 落地

| 图中模块 | Obsidian 对应 | 执行方式 |
|---|---|---|
| 编辑/预览双栏 | Split right + Linked pane | 原生工作区 |
| Outline | Outline 核心插件 | 右侧边栏 |
| Properties | 文件属性 | YAML/frontmatter |
| Linked Mentions | Backlinks | 右侧边栏 |
| Code/Data 区块 | Dataview 查询块 | Dataview |
| Callout 紫色重点块 | blockquote / callout CSS | CSS snippet |

### 执行动作

1. 打开任意笔记，右键标签页 → Split right。
2. 左侧设为 Source mode，右侧设为 Reading view。
3. 右侧栏开启 Outline、Backlinks、Properties。
4. 保存工作区布局为 `Reading Workspace`。

## 3. 图三：Canvas 知识画布 / 项目脑图

### 视觉结构

- 中央主节点：Synthesis / 紫晶核心。
- 周围分组：Research、Core Concepts、Applications、Experiments、Outcomes。
- 连线：曲线连接，主路径紫色，辅助路径蓝/绿/金。
- 右侧属性栏：Canvas name、description、tags、status、node style、overview。
- 底部资产区：图片、Prompt、案例、文件卡片。

### Obsidian 落地

| 图中模块 | Obsidian 对应 | 文件/插件 |
|---|---|---|
| 节点与连线 | `.canvas` JSON Canvas | `02_Canvas/AI知识工作流.canvas` |
| 分组框 | Canvas group node | 原生 Canvas |
| 节点样式 | Canvas color + CSS | CSS snippet |
| 属性栏 | 不能完全原生复刻 | 用右侧笔记 `Project Atlas` 替代 |
| 资产卡片 | Canvas file nodes | Markdown/图片文件 |

### 执行动作

1. 打开 `02_Canvas/AI知识工作流.canvas`。
2. 按项目复制一份：`项目名.canvas`。
3. 每个大项目固定 5 类分组：输入、概念、应用、实验、输出。
4. 中央节点链接到该项目的 `Project Atlas` 笔记。

## 4. 图四：Review + AI Assistant 复习界面

### 视觉结构

- 左侧：Review Overview、Learning Streak、Due Cards、Mastery Levels。
- 中间：Question 卡片、答案折叠、Again/Hard/Good/Easy 四按钮。
- 右侧：AI Assistant，含 Chat / Summarize / Connect / Plan。
- 底部：今日进度、TOP Reviewed Topics。

### Obsidian 落地

| 图中模块 | Obsidian 对应 | 执行方式 |
|---|---|---|
| Review Overview | Markdown 统计卡片 | `01-Review-Session.md` |
| Due Cards | Tasks 查询 `#review` | Tasks |
| Question Card | Markdown `details` | 原生 Markdown |
| 四个复习按钮 | 视觉按钮，真实调度需插件 | CSS + 可选 Spaced Repetition 插件 |
| AI Assistant | AI 指令面板 | Copilot/Smart Connections/本地模型 |
| Top Topics | Dataview 查询 tags | Dataview |

### 执行动作

1. 所有要复习的内容用 `#review` 标记。
2. 在每日笔记里按模板写复习卡。
3. 打开 `00_操作台/01-Review-Session.md` 进行复习。
4. 真正的间隔算法可后续接 Spaced Repetition / 自研插件。

## 5. 图五：紫晶主题视觉资产板

> 上一张图里写了 OBS Studio，但这里已纠偏为 Obsidian 的视觉资产板。

### 视觉结构

- 左侧：品牌名、主题说明、色板、按钮、输入框、图标。
- 中间：主界面预览、主题变体、场景预设、完整布局缩略图。
- 右侧：Widgets & Panels、Notifications、Extra Assets。

### Obsidian 落地

| 图中模块 | Obsidian 对应 | 执行方式 |
|---|---|---|
| 色板 | CSS variables | `purple-gemstone-console.css` |
| 按钮状态 | `.pg-button` `.pg-pill` | CSS snippet |
| Widget Panels | Markdown 卡片组件 | `.pg-card` `.pg-panel` |
| Notifications | Callouts / cards | CSS |
| Theme Variations | body class 或复制 CSS 变量 | Style Settings 可调 |
| Extra Assets | SVG / prompt | `assets/` |

### 执行动作

1. 不把它当作界面，而是当作“设计系统母版”。
2. 所有后续页面都从这个母版取颜色、圆角、阴影、组件。
3. 如果做自研主题，就把这张图作为 Figma/主题开发规范。

## 6. 推荐插件清单

### 必装

- Dataview：最近笔记、阅读队列、项目表、标签查询。
- Tasks：复习队列、项目任务、每日任务。
- Calendar：每日笔记入口。
- Templater：动态模板。
- Style Settings：可视化调 CSS 变量。

### 可选增强

- Homepage：启动时打开 Dashboard。
- Commander：把常用命令固定到顶部/侧边。
- QuickAdd：快速捕获。
- Smart Connections / Copilot：右侧 AI 助手。
- Spaced Repetition：真间隔复习。
- Excalidraw：图形卡片与手绘脑图。

## 8. 开发升级路线

### 第 1 阶段：纯美化

- 启用 CSS Snippet。
- 放入 Dashboard、Review、AI Panel、Canvas。
- 建立目录与模板。

### 第 2 阶段：工作区固定

- 保存 4 个 Workspace：Dashboard / Reading / Canvas / Review。
- Commander 固定快捷命令。
- Homepage 指向 Dashboard。

### 第 3 阶段：数据驱动

- 所有项目、书籍、课程加 frontmatter。
- Dashboard 全部用 Dataview/Bases 自动生成。
- `#review` 进入复习队列。

### 第 4 阶段：AI 助手接入

- 本地：Ollama/DeepSeek + Local REST/自研插件。
- 云端：Copilot/Smart Connections/ChatGPT/Hermes。
- 规则：AI 只读 Vault，不直接删除或移动文件。

### 第 5 阶段：自研插件

- 插件名：`purple-gemstone-console-plugin`。
- 功能：右侧 AI 面板、复习按钮、统计状态栏、快速捕获、项目资产浏览器。

---

## 9. 还原限制

- Obsidian 顶部 Tab、左侧 Ribbon、右侧边栏能 CSS 美化，但不能像设计图那样完全自由重排，除非写插件。
- Dashboard 中的图表、插件状态、AI 回复需要插件或 DataviewJS/自研插件接入真实数据。
- 四个复习按钮当前是视觉层，真实间隔算法需要 Spaced Repetition 或自研插件。
- 图中“右侧 AI Chat”如果要求本地模型，需要额外部署模型服务。
