# V5 课程内容验证与多样化升级标准

> 目标：解决课程库“全是纯文字”的问题。课程笔记不能只是一篇长摘要；它应该同时服务浏览、复习、验证、行动和视觉理解。

## 1. 为什么纯文字不够

纯文字总结有价值，但不应是唯一形态：

- 长文本扫描成本高，难以快速找重点；
- 缺少图表/流程会让方法类课程难以复现；
- 缺少复习题会导致看完就忘；
- 缺少来源/核验说明会让可信度不稳定；
- 缺少课程地图会让课程之间无法形成 KnowledgeOS 导航层；
- 设计、运营、编程、心理、学习力等课程需要不同表达形态，不应都长得一样。

结论：**纯文字可以作为深度层，但不能作为唯一交互层。**

## 2. 每门正式课程的最低内容形态

| 形态 | 文件/位置 | 目的 |
|---|---|---|
| 课程总览 | `00_课程总览.md` | 定位、适用人群、模块、学习路线 |
| 来源核验 | `01_素材识别报告.md` / `06_验证与不确定项.md` | 来源、证据、不确定项 |
| 模块总结 | `03_模块总结.md` / 逐节总结 | 保留深度文字内容 |
| 图表/流程 | Mermaid 或 `04_关键图表与课件索引.md` | 降低理解成本 |
| Canvas 地图 | `02_课程地图.canvas` | 模块/概念/实践路径可视化 |
| 复习练习 | `05_复习与检索练习.md` | 主动回忆、Anki、题目 |
| 行动清单 | `07_实操工作流.md` 或任务列表 | 将课程转成可执行步骤 |
| 课程门户 | `00_课程总览.md` 中的卡片/Dataview | 从使用层进入课程 |

## 3. 多样化评分

脚本 `scripts/v5/course_diversity_audit.py` 会检测：

- 图片/附件引用；
- Markdown 表格；
- Mermaid；
- Dataview；
- 任务清单；
- Callout；
- Canvas/白板链接；
- 代码块；
- 复习题/Anki/自测；
- 来源/证据/核验；
- 行动/练习/作业。

等级：

| 分数 | 等级 | 解释 |
|---:|---|---|
| 8+ | rich | 多形态课程资产 |
| 5-7 | mixed | 有一定结构，但仍可增强 |
| 3-4 | thin | 内容偏单薄 |
| 0-2 | text-heavy | 基本纯文字，需要优先升级 |

## 4. 课程类型差异化要求

| 课程类型 | 重点多样化形态 |
|---|---|
| 设计/UI/视觉 | 图片索引、案例卡、前后对比表、作品拆解 Canvas |
| 编程/AI 开发 | 代码块、流程图、实验步骤、错误排查表 |
| 运营/增长 | 漏斗图、指标表、案例库、行动清单 |
| 学习力/记忆 | 复习题、Anki、训练计划、记忆宫殿图 |
| 心理/沟通 | 情境案例、对话脚本、反思题、风险边界 callout |
| 考试/数学 | 题型表、解题流程、错题卡、时间线复习计划 |

## 5. 不做的事

- 不强行给所有课程套同一种模板；
- 不为了美观移动/删除课程正文；
- 不编造不存在的截图、视频关键帧或来源；
- 不把“待人工确认”伪装成已核验；
- 不把正式 vault 内容上传到 GitHub。

## 6. 推荐升级顺序

1. 先审计 20 门课程，找 text-heavy。
2. 先补课程门户和健康报告。
3. 再补复习/行动/图表模板。
4. 对高价值课程生成 Canvas/Mermaid 地图。
5. 对设计/视觉课程优先补图像索引。
6. 对学习力课程优先补复习题和训练计划。
